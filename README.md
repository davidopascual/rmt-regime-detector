# RMT Spectral Research — Time-Varying Covariance Estimation

A rigorous, academically honest implementation of Random Matrix Theory (RMT)
methods for detecting structural breaks in the covariance eigenvalue distribution
of high-dimensional financial returns.

## Overview

This codebase implements the Marcenko-Pastur framework for spectral analysis of
rolling return covariance matrices, with CUSUM-based change-point detection and
crisis event study validation against the S&P 100 universe (2000-2023).

## Theoretical basis

## Quick start

```python
# 1. Install dependencies
# pip install -r requirements.txt

# 2. Download data (once)
from research.core.loader import download_returns
from research.core.universe import SP100_TICKERS
download_returns(SP100_TICKERS)

# 3. Run rolling estimation
from research.core.loader import load_returns, returns_to_numpy
from research.core.estimator import RollingSpectralEstimator

returns = load_returns()
R, dates, tickers = returns_to_numpy(returns)

est = RollingSpectralEstimator(window=252, step=21)
snapshots = est.fit(R)

# 4. CUSUM change-point detection
from research.core.changepoint import cusum_from_snapshots
cusum_result, _ = cusum_from_snapshots(snapshots)

# 5. Validation
from research.analysis.validation import run_full_validation
results = run_full_validation(snapshots, dates=dates)

# 6. All six figures
from research.analysis.plots import all_figures
figs = all_figures(snapshots, cusum_result, results['roc'], dates=dates)
```

## Key Empirical Findings

Results from the S&P 100 universe, 2000–2024, window T=252, step=21.

**Finding 1: Effect B dominance**
At all five measured crisis episodes, noise floor compression
(Effect B: λ+ falling) accounted for 51–63% of log(ρ) elevation,
consistently exceeding the contribution of market factor amplification
(Effect A: λ₁ rising, 37–49%). Financial crises manifest primarily as
correlation compression rather than market factor strengthening.

**Finding 2: Secular structural break**
The CUSUM detector identified a permanent structural break in 2007 that
has not reversed. The post-GFC 2013–2019 period is insufficiently stable
in ρ-space to serve as a CUSUM null distribution, suggesting the pre-GFC
low-correlation regime is a historical artifact rather than a recoverable
baseline.

**Finding 3: Spike proliferation as crisis signature**
Financial crises manifest as spike proliferation — k(t) rises from 13–15
in calm periods to 28 at GFC peak and 37 at COVID peak — rather than bulk
distortion. Bulk KS AUROC of 0.49 confirms the residual bulk becomes more
MP-like, not less, during crises.

**Finding 4: Effective rank collapse**
Effective rank r_eff(t) collapses from ~12 in pre-GFC calm to 2.5 at
COVID peak, quantifying near-complete diversification breakdown. The
condition number reaches ~10,000 at COVID — the sample covariance matrix
is numerically near-singular.

**Finding 5: Regime 1 CUSUM detects GFC**
A CUSUM detector calibrated on the pre-GFC calm (2004–2007) detects the
GFC with a 56 calendar day delay and 2 false alarms, one of which
(2007-10-04) preceded the GFC start marker by 5 days — a borderline
early warning signal.

## Limitations

The following limitations are documented here and in the relevant module
docstrings.

### Mathematical Flag 1 (mp_theory.py, estimator.py)

The BBP detection threshold theta_c and the MP bulk upper edge lambda+ are
distinct quantities that must not be conflated:

  theta_c = sigma^2 * (1 + sqrt(gamma))      [BBP detection threshold]
  lambda+  = sigma^2 * (1 + sqrt(gamma))^2   [MP bulk upper edge]

  Ratio: lambda+ / theta_c = 1 + sqrt(gamma) > 1 always.

A population spike eigenvalue theta is detectable iff theta > theta_c.
Below theta_c, the sample spike eigenvalue merges with the bulk and is
provably undetectable.  The sample spike eigenvalue (when theta > theta_c)
lands ABOVE lambda+, not at theta_c.

### Mathematical Flag 2 (mp_theory.py, estimator.py)

The BBP phase transition theorem [BBP05, Corollary 1.1] is proven for
complex Gaussian (GUE) random matrices.  Extension to:
  (a) real symmetric (GOE) matrices: conjectured but not proven in [BBP05];
  (b) non-Gaussian entries with finite 4th moment: proven by subsequent
      work (Capitaine, Donati-Martin, Feral 2009) under moment conditions;
  (c) real financial returns: HEURISTIC APPLICATION ONLY.

Financial returns are not Gaussian, are autocorrelated (ARCH effects),
and have heavy tails.  The BBP threshold theta_c is used here as an
approximate guideline.  The self-consistent sigma^2 estimator is also
an approximation; it fails near gamma=1 and when many spikes are present.

### Mathematical Flag 3 (changepoint.py)

The recursive CUSUM formula:

  C(t) = max(0, C(t-1) + (x(t) - mu - delta))

is from post-1954 literature (Lorden 1971; Moustakides 1986).  Page's
original Rule 1 [Page54, p. 101] is stated as:

  "Take action at the first n such that S_n - min_{0<=i<=n} S_i > h"

Both formulations are equivalent for detecting an upward shift, but
attributing the recursive form directly to "Page 1954" is imprecise.

Additionally, CUSUM assumes a stationary in-control process.  Financial
time series violate this assumption due to:
  - Volatility clustering (ARCH/GARCH effects)
  - Regime-dependent baseline KS levels
  - Serial correlation from overlapping rolling windows

CUSUM alarm times should be interpreted as indicative, not statistically
rigorous.  All results are sensitive to the calibration period choice.

### Data limitations

- Survivorship bias: SP100_TICKERS contains long-tenured S&P 500 members
  only. Delisted, merged, or declined stocks are underrepresented.
- Yahoo Finance data quality: adjusted close prices may contain corporate
  action errors.  Not suitable for live trading.
- Within-window standardization: the estimator analyzes CORRELATION
  structure, not covariance structure. Heterogeneous volatility across
  assets is removed. sigma^2 estimates are relative to unit-variance
  entries; under the pure-noise null, sigma^2 = 1 after standardization.
- Crisis calendar look-ahead: the event study crisis labels are constructed
  after observing the data. AUROC values are descriptive, not predictive.

## Design decisions

- **ddof=0 standardization**: within each rolling window, each asset column
  is standardized using the window-mean and population std (ddof=0), not the
  sample std (ddof=1). This ensures Tr(S) = d exactly, which is required for
  the MP law normalization. Using ddof=1 gives Tr(S) = d*(T-1)/T, a ~0.4%
  bias at T=252 that violates the exact MP normalization.

- **Aspect ratio recomputed per window**: gamma(t) = d(t)/T, where d(t) is
  the number of assets with no NaN in window t. This can vary if assets
  are temporarily missing.

- **Self-consistent sigma^2**: the noise floor is estimated by iterating
  the bulk mean (excluding eigenvalues above the current lambda+ estimate)
  to convergence. Initial value: mean of all eigenvalues.
